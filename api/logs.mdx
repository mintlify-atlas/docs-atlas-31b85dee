The logs API exposes ContabilidadISV's system audit trail (*bitácora del sistema*). Only users with `role='admin'` can access either endpoint. Log entries are formatted for human readability before being returned: raw action codes are translated to friendly Spanish titles, dates are localised to Honduras time (UTC-6), and each entry carries a `category` and `priority` label suitable for display in the admin dashboard.

<Warning>
  Both `GET /api/logs` and `POST /api/logs` require admin role. A `403`
  response is returned for any caller with `role='user'`.
</Warning>

<Note>
  Entries with `action = 'api_request'` or `action = 'API_REQUEST'` are
  automatically filtered out of `GET /api/logs` responses. Only meaningful
  business events (logins, consolidation changes, user management, etc.) are
  returned.
</Note>

## GET /api/logs

Retrieves a paginated list of audit log entries, optionally filtered by user
or action type.

**Query parameters**

<ParamField query="limit" type="number">
  Maximum number of entries to return. Defaults to **100**.
</ParamField>

<ParamField query="userId" type="number">
  Filter to entries generated by a specific user (numeric user ID).
</ParamField>

<ParamField query="action" type="string">
  Filter to entries with a specific action code (e.g. `"USER_LOGIN"`,
  `"consolidacion_created"`).
</ParamField>

**Response**

<ResponseField name="success" type="boolean">
  Always `true` on a successful response.
</ResponseField>

<ResponseField name="logs" type="array">
  Array of formatted log entry objects. Each entry contains:

  <Expandable title="log entry fields">
    <ResponseField name="id" type="number">
      Auto-incremented log record ID.
    </ResponseField>
    <ResponseField name="username" type="string">
      Username of the actor, or `"Sistema"` for system-generated events.
    </ResponseField>
    <ResponseField name="action" type="string">
      Raw action code stored in the database (e.g. `"USER_LOGIN"`,
      `"consolidacion_created"`).
    </ResponseField>
    <ResponseField name="description" type="string">
      Detailed description of the event recorded at the time it occurred.
    </ResponseField>
    <ResponseField name="ip_address" type="string">
      Client IP address at the time of the event.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp stored in the database.
    </ResponseField>
    <ResponseField name="formatted_title" type="string">
      Human-readable action title in Spanish (e.g. `"Inicio de sesión de
      usuario"`).
    </ResponseField>
    <ResponseField name="formatted_message" type="string">
      The `description` field, or `"Sin detalles adicionales"` if empty.
    </ResponseField>
    <ResponseField name="category" type="string">
      Event category — see the categories table below.
    </ResponseField>
    <ResponseField name="priority" type="string">
      Event priority — `"normal"`, `"importante"`, or `"critico"`.
    </ResponseField>
    <ResponseField name="friendly_date" type="string">
      Localised date and time formatted as `DD/MM/YYYY HH:MM:SS AM/PM`
      (Honduras, UTC-6 equivalent).
    </ResponseField>
    <ResponseField name="location_info" type="string">
      `"Desde: <ip_address>"`, or `"Sistema interno"` when no IP is recorded.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="statistics" type="object">
  Aggregate statistics for the returned (filtered) log set.

  <Expandable title="statistics fields">
    <ResponseField name="total" type="number">
      Total entries after filtering out `api_request` actions.
    </ResponseField>
    <ResponseField name="today" type="number">
      Entries created today.
    </ResponseField>
    <ResponseField name="thisWeek" type="number">
      Entries created in the current week.
    </ResponseField>
    <ResponseField name="byCategory" type="object">
      Count of entries per action category.
    </ResponseField>
    <ResponseField name="byPriority" type="object">
      Count of entries per priority level (keys: `"normal"`, `"importante"`,
      `"critico"`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number">
  Total count of log entries in the response array (mirrors
  `statistics.total`).
</ResponseField>

```bash
# Last 100 entries (default)
curl -X GET "https://your-host/api/logs" \
  -H "Authorization: Bearer <admin-token>"

# Last 50 login events for user ID 3
curl -X GET "https://your-host/api/logs?limit=50&userId=3&action=USER_LOGIN" \
  -H "Authorization: Bearer <admin-token>"
```

## POST /api/logs

Allows an admin to manually insert a log entry — useful for recording
out-of-band administrative actions or annotating the audit trail. **Admin only.**

**Request body**

<ParamField body="action" type="string" required>
  Action code to record (e.g. `"manual_audit"`, `"database_backup"`). Will be
  stored verbatim and mapped to a category/priority using the same rules as
  system-generated events.
</ParamField>

<ParamField body="description" type="string" required>
  Human-readable description of the event being logged.
</ParamField>

**Response** — `201 Created`

<ResponseField name="success" type="boolean">
  `true` on successful creation.
</ResponseField>

<ResponseField name="message" type="string">
  `"Log creado exitosamente"`.
</ResponseField>

<ResponseField name="log" type="object">
  The newly created log entry in fully formatted form (all fields from `GET
  /api/logs` plus `id` and `created_at`).
</ResponseField>

```bash
curl -X POST https://your-host/api/logs \
  -H "Authorization: Bearer <admin-token>" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "manual_audit",
    "description": "Revisión manual de consolidaciones Q2 completada por administrador."
  }'
```

```json
{
  "success": true,
  "message": "Log creado exitosamente",
  "log": {
    "id": 1025,
    "username": "admin",
    "action": "manual_audit",
    "description": "Revisión manual de consolidaciones Q2 completada por administrador.",
    "ip_address": "10.0.0.1",
    "created_at": "2024-07-10T21:00:00.000Z",
    "formatted_title": "Manual Audit",
    "formatted_message": "Revisión manual de consolidaciones Q2 completada por administrador.",
    "category": "Sistema",
    "priority": "importante",
    "friendly_date": "10/07/2024 3:00:00 PM",
    "location_info": "Desde: 10.0.0.1"
  }
}
```

## Log categories and priorities

The formatting layer maps raw action codes to categories and priorities using
the rules shown in the tables below.

### Categories

| Category | Triggered by |
|----------|-------------|
| **Inicio de Sesión** | `USER_LOGIN`, `login`, or `api_request` containing `"login"` |
| **Cierre de Sesión** | `USER_LOGOUT`, `logout`, or `api_request` containing `"logout"` |
| **Contabilidad** | Actions containing `"consolidacion"` |
| **Gestión de Usuarios** | Actions containing `"user"` or requests to `/users` |
| **Gestión de Clientes** | Actions containing `"cliente"` |
| **Reportes** | Actions containing `"reporte"` or requests to `/report` |
| **Sistema** | `SYSTEM_INIT` or actions containing `"system"` |
| **Errores** | `error` action code |

### Priorities

| Priority | Label | Used when |
|----------|-------|-----------|
| `normal` | Standard event | Logouts, user lookups, client lookups, reports |
| `importante` | Notable event | Logins, consolidation changes, system init |
| `critico` | Critical event | Error events (`action = "error"`) |
